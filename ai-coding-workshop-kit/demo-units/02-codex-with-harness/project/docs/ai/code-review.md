# AI差分レビュー観点

現在の差分を、以下の観点でレビューする。

## 1. 仕様充足

- 仕様書どおりに `GET /discount?amount=...` が追加されているか。
- rank と discountRate が正しく返るか。
- `amount` がレスポンスに含まれているか。

## 2. 境界値

以下の境界値が正しくテストされているか。

- 0
- 9999
- 10000
- 49999
- 50000
- 99999
- 100000

## 3. 異常系

以下が400エラーになるか。

- amount未指定
- 数値ではないamount
- 負数
- 小数

## 4. アーキテクチャ

- business logic が `services/` に置かれているか。
- controller が request / response 処理に集中しているか。
- 既存の `/tax` API の構成と大きくズレていないか。

## 5. 互換性

- 既存の `/health` と `/tax` の挙動を壊していないか。
- 既存レスポンス形式を勝手に変えていないか。

## 6. 保守性

- 実装が過度に複雑ではないか。
- 不要な抽象化をしていないか。
- 命名が分かりやすいか。

## 7. 安全性

- 依存packageを勝手に追加していないか。
- `.env` や秘密情報を読んでいないか。
- 無関係なファイルを変更していないか。
