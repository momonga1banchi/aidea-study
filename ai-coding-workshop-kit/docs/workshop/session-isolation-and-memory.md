# セッション分離とメモリ設定について

## なぜ分離が必要か

悪い例と良い例を同じプロジェクト・同じセッションで実施すると、AIが前の情報を利用してしまう可能性がある。

たとえば、悪い例でAIが会員ランク仕様を見てしまうと、良い例との比較が曖昧になる。  
また、良い例の仕様書を読んだ後に悪い例を実施すると、AIがその知識を使ってしまう可能性がある。

そのため、このキットでは以下の方針を取る。

- 悪い例と良い例でディレクトリを分ける
- Codex / Claudeで開くプロジェクトをデモ単位ごとに分ける
- 講師用プロンプトはAIに見えない場所に置く
- メモリ機能は原則オフにする
- デモごとに新しいセッションを使う

## Codex向け

各Codex用プロジェクトには、以下を配置している。

```text
.codex/config.toml
```

このファイルでは、メモリを使わない前提の設定にしている。

後でメモリを有効化したい場合の例は、以下に置いている。

```text
.codex/config.enable-memory.example.toml
```

勉強会中は、原則としてメモリ無効のまま実施する。

## Claude向け

各Claude用プロジェクトには、以下を配置している。

```text
.claude/settings.json
```

このファイルでは、自動メモリを使わない前提の設定にしている。

後でメモリを有効化したい場合の例は、以下に置いている。

```text
.claude/settings.enable-memory.example.json
```

## 運用上の注意

- デモごとに新しいセッションを作る。
- 悪い例と良い例を同じセッションで続けない。
- Codex / Claudeに開かせるのは必ず `project/` のみ。
- 親ディレクトリを開かない。
- 講師用プロンプトをAIに読ませない。
- デモ後は `scripts/reset-demo-projects.sh` で初期状態に戻す。
