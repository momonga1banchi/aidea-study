# 11. flow-rebuild版 スライド修正案

対象: `aidea-workshop-20260626-flow-rebuild/`

この文書は、2026-06-16の追加フィードバックを受けた修正案である。まだPPTX、生成スクリプト、デモ資料には反映しない。

## 1. 修正対象の問題

今回の指摘は、次の4点に分けて扱う。

1. `schemaなし` という表現が分かりにくい。
2. L1〜L5の説明文がコンソール風表示と近く、視認性が悪い。
3. 全デモで同じ「送料無料条件 5,000円 -> 7,000円」の話が続いていることが弱く、途中で何の話か分からなくなる。
4. 各デモで「具体的にAIへ何を指示するのか」「プロジェクトに何が増えるのか」「その中身は何か」がスライド上で不足している。

## 2. `schema` 表現の修正

### 問題

スライド上の `schemaなし` は、参加者にDBスキーマやJSON Schemaを連想させる可能性がある。

このデモで言いたいことは、DBではなく **APIレスポンス形式の自動チェックがない** ことである。

### 修正方針

参加者向けスライドでは `schema` を原則として使わない。

| 現在の表現 | 修正後 |
|---|---|
| schemaなし | APIレスポンス形式チェックなし |
| schema | レスポンス形式チェック |
| response schema | レスポンス形式 |
| schema sensor | レスポンス形式センサー |

ただし、ターミナルログや `npm run sensor:schema` のようなコマンド名では `schema` が出る。この場合は、横に注釈を置く。

例:

```text
| schema | green |   # APIレスポンス形式チェック
```

### 具体的な修正箇所

- Slide 9: `schema` -> `レスポンス形式チェック`
- Slide 13: `testsなし / schemaなし / 構造チェックなし / 判断記録なし`
  - 修正後: `テストなし / APIレスポンス形式チェックなし / 構造チェックなし / 判断記録なし`
- Slide 14: `response schema`
  - 修正後: `レスポンス形式`
- Slide 17/18/20/22: `schema` を参加者向けには `レスポンス形式チェック` と表記
- Slide 19/23のログ表示では、必要に応じて `schema = APIレスポンス形式` の注釈を付ける
- `concept-primer.md` も同じ用語へ寄せる

## 3. L1〜L5表示位置の修正

### 問題

現在は、次のようなLレベル説明がスライドのサブタイトルとして表示されている。

- `L1: 機能変更はできたかもしれない`
- `L2: 現在地をテストに変える`
- `L3: 動けばOKではなく、保守できる形も見る`
- `L4: Guides / Sensors / Steering`
- `L5: 入口と出口に人間の確認を寄せる`

この位置が、下のコンソール風表示やカードと近く、画面上で読みにくい。デモスライドでは特に、タイトル、L説明、コンソールが縦に詰まっている。

### 修正方針

Lレベル説明をサブタイトルから外す。

代わりに、次の2つへ分ける。

1. 画面下部の進捗バー: L0〜L5の現在地だけを示す。
2. 右上の小さなバッジ: `現在地: L2 既存挙動を固定` のように短く示す。

### 新しい配置ルール

デモスライドの共通レイアウトを固定する。

```text
上部  44-135px: タイトルだけ
右上  44-88px : 現在地バッジ
中央 170-520px: コンソール / 指示カード / 増えたもの
下部 600-652px: L0-L5進捗バー
```

デモスライドでは、サブタイトルは使わない。L説明は本文カードに入れる。

### Lバッジの文言

| レベル | バッジ文言 |
|---:|---|
| L1 | 現在地: L1 証拠なしの変更 |
| L2 | 現在地: L2 既存挙動を固定 |
| L3 | 現在地: L3 構造もチェック |
| L4 | 現在地: L4 リリース候補 |
| L5 | 現在地: L5 介入点を圧縮 |

これにより、長い説明文がコンソールに被らない。

## 4. 全デモで共通のお題を見失わない設計

### 問題

L1からL5まで同じ「送料無料条件変更」の話が続いているにもかかわらず、スライドによっては途中から別の話に見える。

参加者には次の流れが常に見えている必要がある。

> お題はずっと同じ。送料無料条件を5,000円から7,000円へ変更したい。
> 各デモは、その変更を安全に受け入れるために足りないものを1つずつ増やしている。

### 修正方針

Demo 1〜5の各開始スライドに、共通の「お題リボン」を置く。

```text
お題は共通: 送料無料条件を5,000円以上から7,000円以上へ変更する
```

さらに、各デモスライドに次の3項目を固定表示する。

```text
今回の指示
今回増やすもの
この時点でまだ足りないもの
```

これにより、デモが切り替わっても「同じ変更を、だんだん安全にしている」と分かる。

## 6. 生成スクリプト上の修正方針

対象:

```text
aidea-workshop-20260626-flow-rebuild/scripts/build-flow-rebuild-deck.mjs
```

### 6.1 ヘッダー構造

現在の `subtitle` をデモスライドでは使わない。

代わりに次を追加する。

```js
addLevelBadge(slide, level, label)
addCaseRibbon(slide)
```

`addCaseRibbon` はDemo 1〜5の開始スライドに表示する。

```text
お題は共通: 送料無料条件 5,000円以上 -> 7,000円以上
```

### 6.2 デモスライドテンプレート

Demo 1〜5の開始スライドは、以下のテンプレートに寄せる。

```text
左上: 今回の指示
右上: 今回増やすもの
左下: 実行ログまたはファイルツリー
右下: まだ足りないもの / ここで達成したこと
下部: L0-L5進捗バー
```

### 6.3 文言置換

スライド本文では次を置換する。

```text
schema -> APIレスポンス形式チェック
response schema -> レスポンス形式
schema sensor -> レスポンス形式センサー
```

ただし、ターミナルログ内の `schema` は実コマンド名として残してよい。その場合は横に注釈を置く。

## 7. 検収観点

修正後は、既存検査に加えて次を見る。

### 7.1 見た目

- L1〜L5の説明がコンソール風ボックスと重なっていない。
- デモスライドのタイトル下に長いサブタイトルがない。
- L0〜L5進捗バーは下部に固定され、本文を邪魔しない。

### 7.2 用語

- 参加者向け本文に `schemaなし` が残っていない。
- `schema` が出る場合は、APIレスポンス形式チェックだと分かる注釈がある。

### 7.3 ストーリー

- Demo 1〜5で同じ送料無料条件変更が続いていると分かる。
- 各デモに「今回の指示」がある。
- 各デモに「今回増やすもの」がある。
- 各デモに「まだ足りないもの」または「ここで達成したこと」がある。
- Demo 4で、ワークショップ上のリリース候補に到達したことが明示されている。
